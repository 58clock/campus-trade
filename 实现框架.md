## 一、技术栈总览

| 层级 | 技术 | 说明 |
|------|------|------|
| **后端框架** | Spring Boot 3.x | Java 17+，RESTful API |
| **安全认证** | Spring Security + JWT | 令牌认证、角色权限 |
| **数据库** | MySQL 8.0 | 关系型数据存储 |

| **状态管理** | Pinia | Vue 3 官方推荐 |
| **HTTP 客户端** | Axios | 请求拦截、令牌携带 |
| **图表库** | ECharts 5 | 后台统计图表 |

## 四、各模块实现逻辑

### 模块一：用户系统

**注册流程：**
```
前端表单 → POST /api/auth/register → 校验字段 → BCrypt 加密密码
→ 存入 user 表 → 返回成功
```
- 邮箱/手机号唯一性校验
- 密码强度校验（长度 >= 6，含字母+数字）
- 密码 BCrypt 加密存储

**登录流程：**
```
POST /api/auth/login → 查询用户 → 检查是否被封禁/冻结
→ 校验密码 → 检查失败次数（Redis 计数器）
→ 生成 JWT（含 userId + role，有效期 24h）→ 返回 token
```
- 登录失败 5 次冻结账号 30 分钟（用 Redis 记录失败计数，带 TTL）
- JWT 结构：`{ userId: 1, role: "USER", exp: ... }`

**忘记密码：**
```
输入邮箱 → 生成 6 位验证码存 Redis（5分钟有效）
→ 控制台打印验证码（模拟发送邮件）
→ 用户输入验证码 + 新密码 → 校验通过 → 更新密码
```

**个人信息管理：**
- `GET /api/user/profile` — 获取个人资料
- `PUT /api/user/profile` — 更新昵称、学校、联系方式
- `POST /api/user/avatar` — Multipart 上传头像，存本地 `/uploads/avatar/`

**Spring Security 配置关键点：**
```java
// SecurityConfig.java
// 放行: /api/auth/**, /api/products (GET), /swagger-ui/**
// 其余需要认证
// 管理员接口需要 @PreAuthorize("hasRole('ADMIN')")
```

**我的发布列表：**


**编辑 / 下架 / 删除：**
- `PUT /api/products/{id}` — 编辑（仅作者可操作）
- `PUT /api/products/{id}/off-shelf` — 下架（改状态）
- `DELETE /api/products/{id}` — 软删除（改状态为 DELETED）

**搜索与筛选（核心查询）：**
```
GET /api/products?keyword=手机&category=ELECTRONICS&minPrice=100&maxPrice=500&sort=price_asc&page=1&size=12
```
后端 MyBatis-Plus 动态条件构造器：
```java
LambdaQueryWrapper<Product> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Product::getStatus, "ON_SALE");
if (keyword != null) wrapper.and(w -> w.like(Product::getTitle, keyword).or().like(Product::getDescription, keyword));
if (category != null) wrapper.eq(Product::getCategory, category);
if (minPrice != null) wrapper.ge(Product::getPrice, minPrice);
if (maxPrice != null) wrapper.le(Product::getPrice, maxPrice);
// 排序：price_asc / price_desc / newest
Page<Product> page = new Page<>(pageNum, pageSize);
productMapper.selectPage(page, wrapper);
```

### 模块八：开放 Skill

推荐选**智能推荐 Skill**（数据容易构造，效果好展示）：

```
POST /api/skills/recommend
输入: { "userId": 1, "limit": 10 }
输出: { "recommendations": [ { "productId": ..., "title": ..., "score": 0.95 }, ... ] }
```

**实现原理（简单协同过滤或基于内容的推荐）：**
1. 获取用户最近浏览/搜索的商品分类
2. 在同分类下按浏览量 + 发布时间加权排序
3. 排除用户自己的商品和已下架商品
4. 返回 Top N

```java
@Service
public class RecommendSkill {
    public List<ProductVO> recommend(Long userId, int limit) {
        // 1. 查用户最近浏览记录（或直接用最近查看商品分类）
        // 2. 同分类热门商品推荐
        // 3. 简单实现：同分类按浏览量倒序 + 排除已看
        return productMapper.selectList(new LambdaQueryWrapper<Product>()
            .eq(Product::getStatus, "ON_SALE")
            .ne(Product::getUserId, userId)
            .orderByDesc(Product::getViewCount)
            .last("LIMIT " + limit))
            .stream().map(this::toVO).toList();
    }
}
```

## 六、统一响应格式

```json
// 成功
{ "code": 200, "message": "success", "data": { ... } }
// 分页
{ "code": 200, "message": "success", "data": { "records": [...], "total": 100, "page": 1, "size": 12 } }
// 失败
{ "code": 400, "message": "用户名已存在", "data": null }
```

```java
// Result.java
@Data
@AllArgsConstructor
public class Result<T> {
    private int code;
    private String message;
    private T data;

    public static <T> Result<T> ok(T data) { return new Result<>(200, "success", data); }
    public static <T> Result<T> fail(String msg) { return new Result<>(400, msg, null); }
}
```
